## 9.2 配置方針の説明

### コア概念はどこに？
`src/core/model/` — UI非依存。ブラウザAPIにも依存しない純粋な型定義とロジック。
ここが全体の基盤であり、最も重要かつテスト密度が高い場所。

### フィーチャーモジュールはどこに？
`src/features/` — Phase 2以降で追加される機能。
各フィーチャーは `src/core/` に依存するが、フィーチャー間は疎結合。
FORMやグラ���などPKC1で巨大化した機能はここに隔離する。

### ビルドスクリプトはどこに？
`build/` — Vite設定。単一HTMLを生成するためのバンドル設定。
`npm run build` → `dist/pkc2.html` を出力。

### ドキュメントはどこに？
`docs/` — 設計文書(`design/`)、操作シナリオ(`scenarios/`)、開発者向け(`dev/`)に分離。
計画文書(`planning/`)は本レポート。
requirements(`requirements/`)は一次資料として保全。

### 利用手順系ドキュメントはどこに？
`docs/scenarios/` — 操作順序依存バグ対策の核心。
各シナリオはステップバイステップで操作と期待結果を記述。
テスト（`tests/scenarios/`）と対になる。
