# 04. PKC2 継承判断（A/B/C/D区分）

現行PKCの機能・概念を、PKC2での扱い4区分に振り分ける。

## 判断基準

| 基準 | 説明 |
|------|------|
| Req寄与 | requirements への寄与度（高/中/低） |
| UX寄与 | UXへの寄与度（高/中/低） |
| コア整合 | コア概念との整合性���高/中/低） |
| HTML適合 | 単一HTMLとして持つ意味（高/中/低） |
| 順序リスク | 操作順序依存や複雑化の危険性（高/中/低） |
| 後追加 | 後から追加可能か（容易/可能/困難） |
