# PKC2 ユーザーマニュアル

PKC2 は「単一 HTML で動作するローカル完結型の知識コンテナ」です。このマニュアルは、PKC2 を日常的に使うための手引きです。

## このマニュアルの読み方

このマニュアルには 2 つの形があります。どちらも内容は同じです。

- **Markdown 版**: `docs/manual/*.md` — GitHub やエディタで直接読みたい方向け
- **PKC2 HTML 版**: `PKC2-Extensions/pkc2-manual.html` — PKC2 自身で開いて読みたい方向け（readonly。Rehydrate でご自身のワークスペースに取り込むこともできます）

## 対象読者

- PKC2 を個人の知識管理ツールとして使いたい方
- 開発者ではなく、ブラウザで HTML ファイルを開いて使う一般の利用者

PKC2 の内部設計や実装について知りたい開発者の方は、`docs/planning/00_index.md` を参照してください。

## 最近の UX 改善（2026-04-22 時点）

利用者から報告のあった UI の引っかかり（UI continuity wave）を一巡で解消した分をマニュアルに反映済みです。主な改善点は:

- **スクロール位置の保持** — TEXTLOG のチェックリストや HTML 添付の sandbox 許可 checkbox を切り替えてもペインが上端に戻らなくなりました（[05 日常操作 > クイック更新](05_日常操作.md#クイック更新)）
- **Recent パネルの折りたたみ保持** — 畳んだ状態が再描画後も維持されます（[03 画面とビュー > Recent エントリパネル](03_画面とビュー.md#recent-エントリパネル)）
- **Storage Profile のアクセス** — メニューから開けるストレージ使用量のオーバーレイ。閉じるまで維持されます（[03 画面とビュー > Storage Profile](03_画面とビュー.md#storage-profileストレージプロファイル)）
- **フォルダが勝手に全開しない** — 畳んだ意図が尊重され、外部ジャンプに近い一部経路だけ必要な先祖フォルダを展開します（[03 画面とビュー > フォルダの自動展開の扱い](03_画面とビュー.md#フォルダの自動展開の扱い)）
- **別ウィンドウの最小ショートカット** — ダブルクリックで開いた編集ウィンドウで `Ctrl+S` / `Cmd+S` と `Escape` が使えます（まだこれだけです。[06 キーボードショートカット](06_キーボードショートカット.md#別ウィンドウentry-windowでの最小ショートカット)）

過去に経験していた症状に心当たりがある場合は [09 トラブルシューティング](09_トラブルシューティングと用語集.md) の「過去の症状」節も合わせて参照してください。

> 開発側の詳細は `docs/planning/HANDOVER_FINAL.md §23`（UI Continuity Wave Closure）を参照。artifact size / dead-export / child window shortcut の slice 2 以降は別トラックで進めており、本マニュアルには実装後に反映します。

## 見本エントリについて

このマニュアルの HTML 版には、PKC2 の各 archetype（エントリの種類）の「見本エントリ」が同梱されています。左サイドバーの **見本** フォルダから確認できます。

- 期日付き / 期日なしの Todo
- タイムスタンプ付きの TextLog
- Form の入力サンプル
- 子エントリを持つ Folder
- 画像とテキストの Attachment

Calendar ビューや Kanban ビューに切り替えると、Todo 見本がそれぞれの表示形式でどう見えるかを確認できます。
