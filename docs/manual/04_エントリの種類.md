# エントリの種類（Archetype）

PKC2 では、すべてのデータ単位を **Entry**（エントリ）と呼びます。各エントリには **archetype**（種別）があり、それによって編集画面や表示のされ方が変わります。

![エントリ編集画面](asset:04_entry-editor)

## Archetype 一覧

| Archetype | 用途 | 本文の中身 |
|-----------|------|-----------|
| `text` | メモ・ノート・ドキュメント | Markdown テキスト |
| `textlog` | 日誌・ログ・議事録 | タイムスタンプ付きの追記型テキスト |
| `todo` | タスク管理 | ステータス・期日・説明（Markdown）の JSON |
| `form` | 構造化記録 | name / note / checked の固定フィールド JSON |
| `attachment` | ファイル添付 | ファイル名・MIME・サイズ・アセットキーの JSON |
| `folder` | 階層整理 | フォルダ説明（Markdown、任意） |
| `generic` | 汎用・フォールバック | Markdown として扱われる |
| `opaque` | 不明な旧形式 | Markdown としてそのまま表示 |

## text — メモ・ノート

最も汎用的な archetype です。Markdown で自由に書けます。

- **使いどころ**: アイデア、議事録、ドキュメント、リファレンス
- **レンダリング**: Markdown → HTML（見出し、リスト、リンク、コードブロック、表、画像参照）
- **コードハイライト**: fenced code block で言語名を付けるとハイライトされます
- **画像参照**: `![説明](asset:アセットキー)` 形式で attachment エントリの画像を埋め込めます
- **非画像の添付参照**: `[ラベル](asset:アセットキー)` でチップとしてリンクできます

### エントリの参照と埋め込み

text の本文には、他のエントリへのリンクや埋め込みを書けます。

- **リンクのみ**: `entry:<lid>` または `[ラベル](entry:<lid>)` — クリックでそのエントリに移動
- **埋め込み（transclusion）**: `![](entry:<lid>)` — 表示時に参照先の内容がそのまま展開されます（TODO のステータスカード、別 TEXT の本文など）

埋め込みはワンクリックで遷移しなくても内容を視界に入れたいときに便利です。depth 制限があり、埋め込みの中でさらに埋め込むことはできません（循環や深い入れ子を自動で遮断）。

## textlog — 日誌・ログ

追記専用の日誌です。

- **使いどころ**: 作業ログ、学習メモ、日記、会話ログ
- **操作**: `Ctrl+Enter` / `Cmd+Enter` で新しいエントリが追記されます
- **タイムスタンプ**: 各追記に自動的に日時が付きます
- **★フラグ**: 個別の追記に星印を付けて重要度を示せます
- **編集・削除**: 追記単位で編集・削除できます
- **日付ごとのグルーピング**: 詳細ビューでは追記が日付ごとに区切って表示されます
- **検索・置換**: 編集モード中、各ログ行の右側にある `🔎` ボタンで、そのログ行の本文だけを対象にした Find & Replace ダイアログを開けます（1 件ずつ単位、詳細は [05 日常操作](05_日常操作.md) 参照）

### TEXT への変換

TEXTLOG のログを選んで、1 つの Markdown ドキュメントにまとめることができます。選択モードからプレビュー → 作成の 2 ステップで、新しい TEXT エントリが生まれます。詳しくは [05 日常操作](05_日常操作.md) の「TEXT と TEXTLOG を変換する」を参照してください。

## todo — タスク管理

ステータス・期日・説明を持つタスクです。

- **使いどころ**: 個人のタスク、買い物リスト、To Do
- **ステータス**: `open`（未完了）/ `done`（完了）
- **期日**: `YYYY-MM-DD` 形式（任意）
- **期限切れ表示**: 期日が過ぎた未完了タスクは赤く強調されます
- **アーカイブ**: 完了した Todo をアーカイブすると通常の一覧から隠れます
- **Calendar ビュー**: 期日付き Todo が月カレンダーに配置されます
- **Kanban ビュー**: ステータス別に 2 列で表示されます

### 説明は Markdown

`description` フィールドは **Markdown としてレンダリング** されます。単なるタスク名だけでなく、作業手順や補足情報、他エントリへのリンク（`entry:<lid>`）や添付画像（`asset:<key>`）を書いておくことができます。

### Todo を他エントリに埋め込む

text や folder の本文に `![](entry:<todo-lid>)` と書くと、その場所に TODO の状態カード（ステータス / 期日 / 説明）が展開されます。進捗の可視化やダッシュボードのように使えます。

### Todo の本文形式

Todo の本文は次のような JSON で保存されます。

```json
{
  "status": "open",
  "description": "買い物に行く",
  "date": "2026-04-15",
  "archived": false
}
```

UI 上では編集画面で入力するだけで、この JSON は自動的に生成されます。

## form — 構造化記録

固定フィールドの簡易フォームです。

- **使いどころ**: 連絡先、チェック項目、決まった形式のメモ
- **フィールド**:
  - `name` — 名前（テキスト）
  - `note` — メモ（テキスト）
  - `checked` — チェックボックス（真偽値）
- **設計の割り切り**: 自由なスキーマ設計はできません。固定の 3 フィールドだけです

自由な構造化データが必要な場合は text archetype で Markdown テーブルや YAML を書く方が柔軟です。

## attachment — ファイル添付

画像・PDF・動画・音声・テキストファイルなどを保存します。

- **使いどころ**: 参考資料、画像、エビデンス、PDF
- **作成方法**:
  - ツールバーの **+ File** ボタン
  - 画面下部の常設ドロップゾーンにドラッグ&ドロップ
  - サイドバーやエディタに直接ドロップ
  - `Ctrl+V` でクリップボード画像を貼り付け
- **格納方法**: ファイル本体は `container.assets` に base64 で、メタデータは Entry の `body` に保存されます
- **自動配置**: 編集中のフォルダ配下の `ASSETS` サブフォルダに自動的に収納されます（なければ自動作成）
- **プレビュー**:
  - 画像 (png/jpeg/gif/webp) — インライン表示
  - PDF — 埋め込みプレビュー
  - 音声・動画 — ブラウザの標準プレイヤー
  - HTML / SVG — サンドボックス iframe で表示（権限はユーザーが調整可能）
- **サイズ警告**:
  - 1 MB 未満: 警告なし
  - 1 MB 以上: 軽い警告
  - 5 MB 以上: 重めの警告（外部ストレージの検討を推奨）
  - これらは制限ではなく情報提供のみです

### サンドボックス設定

HTML / SVG 添付のプレビューは iframe の `sandbox` 属性で隔離されます。コンテナ全体の既定は `strict`（`allow-same-origin` のみ）ですが、meta パネルから `relaxed`（+ `allow-scripts`, `allow-forms`）に切り替えられます。per-entry で個別に許可を追加することもでき、per-entry の設定は container 既定より優先されます。

## folder — 階層整理

他のエントリを子として束ねるフォルダです。

- **使いどころ**: プロジェクト、トピック、カテゴリ
- **サイドバー**: フォルダツリーとして階層表示
- **開閉**: フォルダ見出しのクリックで展開 / 折りたたみ
- **子エントリ**: structural relation でフォルダと子エントリが結ばれます
- **説明（Markdown）**: folder 自身も本文を持ちます。Markdown でプロジェクト説明やダッシュボードを書けます
- **埋め込みの受け皿**: folder 説明の Markdown に `![](entry:<lid>)` を書いて、配下の TODO カードなどを並べることもできます

### ASSETS / TODOS の自動フォルダ

添付を貼り付けたときや Todo を作ったときに、編集コンテキストに応じて `ASSETS` / `TODOS` という子フォルダが自動作成され、そこに配置されます。ルート直下では自動作成されません（散らからないための割り切り）。

## generic / opaque

明示的な archetype を持たないエントリや、互換性のための旧形式に使われます。通常はユーザーが直接作る必要はありません。

## 見本エントリを見てみる

PKC2 マニュアル HTML 版の左サイドバーにある **見本** フォルダには、各 archetype の実動作するエントリが入っています。実際に開いて、編集画面や Calendar / Kanban 上での見え方を確認してみてください。
