# エントリの種類（Archetype）

PKC2 では、すべてのデータ単位を **Entry**（エントリ）と呼びます。各エントリには **archetype**（種別）があり、それによって編集画面や表示のされ方が変わります。

![エントリ編集画面](asset:04_entry-editor)

## Archetype 一覧

| Archetype | 用途 | 本文の中身 |
|-----------|------|-----------|
| `text` | メモ・ノート・ドキュメント | Markdown テキスト |
| `textlog` | 日誌・ログ・議事録 | タイムスタンプ付きの追記型テキスト |
| `todo` | タスク管理 | ステータス・期日・説明の JSON |
| `form` | 構造化記録 | name / note / checked の固定フィールド JSON |
| `attachment` | ファイル添付 | ファイル名・MIME・サイズ・アセットキーの JSON |
| `folder` | 階層整理 | フォルダ説明（任意） |
| `generic` | 汎用・フォールバック | Markdown として扱われる |
| `opaque` | 不明な旧形式 | Markdown としてそのまま表示 |

## text — メモ・ノート

最も汎用的な archetype です。Markdown で自由に書けます。

- **使いどころ**: アイデア、議事録、ドキュメント、リファレンス
- **レンダリング**: Markdown → HTML（見出し、リスト、リンク、コードブロック、表、画像参照）
- **画像参照**: `![説明](asset:アセットキー)` 形式で attachment エントリの画像を埋め込めます
- **非画像の添付参照**: `[ラベル](asset:アセットキー)` でチップとしてリンクできます

## textlog — 日誌・ログ

追記専用の日誌です。

- **使いどころ**: 作業ログ、学習メモ、日記、会話ログ
- **操作**: `Ctrl+Enter` / `Cmd+Enter` で新しいエントリが追記されます
- **タイムスタンプ**: 各追記に自動的に日時が付きます
- **★フラグ**: 個別の追記に星印を付けて重要度を示せます
- **編集・削除**: 追記単位で編集・削除できます

## todo — タスク管理

ステータス・期日・説明を持つタスクです。

- **使いどころ**: 個人のタスク、買い物リスト、To Do
- **ステータス**: `open`（未完了）/ `done`（完了）
- **期日**: `YYYY-MM-DD` 形式（任意）
- **期限切れ表示**: 期日が過ぎた未完了タスクは赤く強調されます
- **アーカイブ**: 完了した Todo をアーカイブすると通常の一覧から隠れます
- **Calendar ビュー**: 期日付き Todo が月カレンダーに配置されます
- **Kanban ビュー**: ステータス別に 2 列で表示されます

### Todo の本文形式

Todo の本文は次のような JSON で保存されます。

```json
{
  "status": "open",
  "description": "買い物に行く",
  "date": "2026-04-15",
  "archived": false
}
```

UI 上では編集画面で入力するだけで、この JSON は自動的に生成されます。

## form — 構造化記録

固定フィールドの簡易フォームです。

- **使いどころ**: 連絡先、チェック項目、決まった形式のメモ
- **フィールド**:
  - `name` — 名前（テキスト）
  - `note` — メモ（テキスト）
  - `checked` — チェックボックス（真偽値）
- **設計の割り切り**: 自由なスキーマ設計はできません。固定の 3 フィールドだけです

自由な構造化データが必要な場合は text archetype で Markdown テーブルや YAML を書く方が柔軟です。

## attachment — ファイル添付

画像・PDF・動画・音声・テキストファイルなどを保存します。

- **使いどころ**: 参考資料、画像、エビデンス、PDF
- **格納方法**: ファイル本体は `container.assets` に base64 で、メタデータは Entry の `body` に保存されます
- **プレビュー**:
  - 画像 (png/jpeg/gif/webp) — インライン表示
  - PDF — 埋め込みプレビュー
  - 音声・動画 — ブラウザの標準プレイヤー
  - HTML / SVG — サンドボックス iframe で表示（権限はユーザーが調整可能）
- **サイズ警告**:
  - 1 MB 未満: 警告なし
  - 1 MB 以上: 軽い警告
  - 5 MB 以上: 重めの警告（外部ストレージの検討を推奨）
  - これらは制限ではなく情報提供のみです

## folder — 階層整理

他のエントリを子として束ねるフォルダです。

- **使いどころ**: プロジェクト、トピック、カテゴリ
- **サイドバー**: フォルダツリーとして階層表示
- **開閉**: フォルダ見出しのクリックで展開 / 折りたたみ
- **子エントリ**: structural relation でフォルダと子エントリが結ばれます

## generic / opaque

明示的な archetype を持たないエントリや、互換性のための旧形式に使われます。通常はユーザーが直接作る必要はありません。

## 見本エントリを見てみる

PKC2 マニュアル HTML 版の左サイドバーにある **見本** フォルダには、各 archetype の実動作するエントリが入っています。実際に開いて、編集画面や Calendar / Kanban 上での見え方を確認してみてください。
