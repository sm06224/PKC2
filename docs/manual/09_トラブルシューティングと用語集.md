# トラブルシューティングと用語集

## トラブルシューティング

### データが消えた・見当たらない

**症状**: 前回作成したエントリが表示されない。

**考えられる原因**:

1. **別のブラウザ / 別の HTML ファイルで開いている**: IndexedDB はブラウザごと・URL ごとに分かれています。同じ HTML ファイルを同じブラウザで開き直してください
2. **ブラウザのサイトデータをクリアした**: ブラウザの履歴・サイトデータ削除で IndexedDB も消えます
3. **プライベートブラウジングで使った**: プライベートモードのデータはタブを閉じると消えます
4. **ファイルのダウンロード先が変わった**: 同じ HTML ファイルでも保存場所が変わると `file:///` URL が変わり、別のストアになります

**対処**:

- データの保険として定期的に Export（特に ZIP Package）でバックアップしてください
- 複数デバイスで使う場合は、ZIP Package を共有ストレージに置いて運用するのが確実です

### Readonly モードで編集できない

**症状**: エントリの編集ボタンが表示されない。

**原因**: Readonly としてエクスポートされた HTML を開いています。

**対処**:

- 編集したい場合は **Rehydrate to Workspace** ボタンを押してください
- Rehydrate すると新しい Container ID が発行され、編集可能なワークスペースとして IndexedDB に保存されます
- 元の HTML ファイル自体は変更されません

### ファイルサイズが大きいと警告が出る

**症状**: ファイル添付時に「このファイルは大きすぎます」的なメッセージ。

**原因と対処**:

- **1 MB 以上**: 軽い警告。操作はブロックされません
- **5 MB 以上**: 重めの警告。Export の HTML サイズに影響します。大きい添付が多い場合は **ZIP Package** でのバックアップを検討してください
- これらは制限ではなく情報提供です。無視して進めることもできます

### Export したら HTML が数十 MB になった

**症状**: 書き出した HTML が重い。

**原因**: 添付ファイルを HTML Full モードで同梱すると、gzip 圧縮しても一定のサイズになります。

**対処**:

- 共有用途なら **HTML Light**（添付実体なし）を使う
- バックアップ用途なら **ZIP Package**（base64 膨張なし）を使う
- 巨大な添付は元ファイルを別管理にする選択も検討

### Import したらデータが全部置き換わった

**症状**: Import 後に元のエントリが消えた。

**原因**: Import は **全置換** です。マージ機能はありません。

**対処**:

- Import 前に必ず Export でバックアップを取る習慣をつけてください
- Confirm Import のダイアログで内容を確認してから確定しましょう

### IndexedDB が壊れた・読み込みエラー

**症状**: PKC2 を開いても初期化が終わらない、エラーが出る。

**対処**:

1. ブラウザのデベロッパーツール（`F12`）を開き、コンソールのエラーメッセージを確認
2. **Clear IDB** ボタンが表示されている場合はそれを使って IndexedDB をクリア（データは消えます）
3. IDB クリア後、HTML 内の pkc-data にフォールバックします。Export 済みのバックアップから Import で復元してください
4. 再発する場合は別のブラウザで試してみてください

### ブラウザのタブを閉じたら保存されなかった気がする

**症状**: 編集中にタブを閉じたら最新の変更が消えた。

**対処**:

- PKC2 は 300 ミリ秒のデバウンスで保存するため、編集直後にタブを閉じると間に合わないことがあります
- 確実に保存したい場合は `Ctrl+S` / `Cmd+S` で明示的に確定してください
- `pagehide` イベントでの強制保存もありますが、OS の強制終了では働きません

### 埋め込み（iframe）で PKC2 が動かない

**症状**: 親ページに iframe で PKC2 を埋め込んだが、表示されない・操作できない。

**対処**:

- iframe の `sandbox` 属性に `allow-scripts allow-same-origin` が含まれているか確認
- `allow-top-navigation` は付けないでください（セキュリティ境界を壊します）
- 親ページから `postMessage` で通信する場合は `origin` を検証してください
- 詳細は開発者向け文書 `docs/planning/14_基盤方針追補_clone_embed_message.md` を参照

## 用語集

### Archetype（アーキタイプ / エントリ種別）

エントリの種類。`text` / `textlog` / `todo` / `form` / `attachment` / `folder` / `generic` / `opaque` の 8 種類があります。archetype ごとに専用のレンダラ（presenter）が本文の見え方を決めます。

### Asset（アセット）

ファイル添付の実体データ。`container.assets` という独立したストアに base64 エンコードされて保存されます。エントリの本文には `asset_key` だけが保存され、実体は分離されています。

### Container（コンテナ）

PKC2 の永続データの最上位の集約。`meta` / `entries` / `relations` / `revisions` / `assets` の 5 要素から構成されます。Export の単位でもあります。

### Entry（エントリ）

データの基本単位。`lid`（ローカル ID）、`title`、`body`、`archetype`、`created_at`、`updated_at` の 6 フィールドを持ちます。旧称は Record でしたが、TypeScript の組み込み型 `Record<K, V>` との衝突を避けて Entry に改名されました。

### Export

Container を HTML または ZIP として書き出す機能。配布・バックアップ・マシン移行に使います。

### HTML Light / HTML Full

HTML Export の 2 つのモード。Light は添付実体を含めず軽量、Full は全データを gzip 圧縮付きで同梱します。

### Import

HTML または ZIP ファイルから Container を読み込む機能。現在のワークスペースを**完全に置き換え**ます。

### IndexedDB（IDB）

ブラウザ内の永続ストレージ API。PKC2 はここに Container と Assets を保存します。ブラウザごと・ドメインごとにストアが分かれます。

### LID（Local ID）

Container 内でユニークな Entry の ID。複数の Container 間で衝突する可能性があるため「ローカル」と呼ばれます。

### pkc-data / pkc-meta / pkc-core

PKC2 HTML の固定スロット。それぞれ Container の JSON・リリースメタデータ・アプリケーションコード（JavaScript バンドル）を格納します。

### Readonly

HTML Export の mutability の 1 つ。開いた時に編集ボタンが表示されず、閲覧専用の UI ポリシーになります。セキュリティ機能ではなく、あくまで「うっかり編集を防ぐ」ガードです。

### Rehydrate（再水和）

Readonly HTML を Editable なワークスペースに昇格させる操作。新しい Container ID が発行され、IndexedDB に保存されます。元の HTML ファイルは変更されません。

### Relation（リレーション）

エントリ同士を結ぶタイプ付きのエッジ。`structural` / `categorical` / `semantic` / `temporal` の 4 種類があります。

### Revision（リビジョン）

エントリの編集前の状態を記録したスナップショット。編集のたびに自動保存され、後から復元できます。

### ZIP Package

`manifest.json` + `container.json` + `assets/` で構成される PKC2 独自の ZIP 形式。添付ファイルを生バイナリで保存するため、最もサイズ効率が良く、Git 管理にも向きます。

### Workspace（ワークスペース）

編集可能な状態の Container を指します。IndexedDB に保存されていて、UI 上で編集・作成・削除ができる状態です。Readonly HTML と対比されます。
