# 画面とビュー

PKC2 の画面構成と、ビューの切り替え方法を説明します。

## 画面全体の構成

PKC2 の画面は大きく次の要素で構成されています。

- **ツールバー**（画面上部）— エントリ作成ボタン、ビュー切替、検索、Export / Import
- **サイドバー**（画面左）— エントリ一覧とフォルダツリー
- **詳細ペイン**（画面中央〜右）— 選択中のエントリの本文
- **情報パネル**（状況に応じて表示）— リビジョン履歴、関連付け、タグなど
- **常設ドロップゾーン**（画面下部）— ファイルをドラッグするだけで attachment として追加できる領域

![Detail ビュー](asset:01_detail-view)

## ペインの表示・非表示

画面を狭くしたいときや、逆に詳細だけに集中したいときは、サイドバーや情報パネルを開閉できます。

- `Ctrl+\` / `Cmd+\` — **左ペイン**（サイドバー）を表示 / 非表示
- `Ctrl+Shift+\` / `Cmd+Shift+\` — **右ペイン**（情報パネル）を表示 / 非表示

同じショートカットをもう一度押すと、元の状態に戻せます。

## サイドバー

サイドバーには、ワークスペース内の全エントリが表示されます。

- **フォルダツリー** — folder archetype のエントリを親とした階層表示
- **エントリの選択** — クリックで選択すると、詳細ペインに表示されます
- **一括選択** — `Ctrl+Click` / `Shift+Click` で複数選択。一括削除やフォルダ移動ができます
- **フォルダの開閉** — フォルダ見出しをクリックで展開・折りたたみ
- **右クリックメニュー** — コンテキストメニューから編集・複製・削除・別ウィンドウ表示などを呼び出せます
- **ドラッグ&ドロップ** — エントリを別フォルダ（またはルート）にドロップして階層を変更できます

複数選択したままドラッグすると、選択中のエントリがまとめて移動します。

### 選択状態の強調

複数選択中のエントリには強調マーカーが付きます。ツールバーには選択件数に応じて一括操作のボタンが現れます。

## フォルダの基本操作

folder archetype のエントリが「フォルダ」としてサイドバーに表示されます。

- フォルダは子エントリを構造的に束ねる入れ物です（`structural` relation）
- フォルダ自身もエントリの 1 種で、説明文（Markdown）を持てます
- 自動生成されるフォルダ（添付を貼り付けたときの `ASSETS`、Todo を作ったときの `TODOS` など）もあります
- フォルダ配下の一括持ち出しは **フォルダスコープ Export**（後述）で行います

![エントリ編集画面](asset:04_entry-editor)

## 検索とフィルタ

サイドバー上部の検索バーから、ワークスペース内を全文検索できます。

- **検索クエリ** — タイトルと本文の両方を対象に部分一致で検索（大文字小文字を区別しません）
- **Archetype フィルタ** — text / todo / form など、特定の種類だけを絞り込む
- **タグフィルタ** — タグ（categorical relation）で絞り込む
- **ソート** — タイトル・作成日時・更新日時の昇順 / 降順
- **アーカイブ表示** — アーカイブ済み Todo の表示・非表示を切り替える
- **フィルタクリア** — すべてのフィルタを一括解除

検索とフィルタは即座に反映され、サイドバーの一覧が更新されます。

## 3 つのビュー

画面上部のビュー切替ボタンで、同じデータを違う見方で表示できます。ビューを切り替えても **選択状態は保持** されます。

### Detail ビュー（詳細ビュー）

標準のビューです。サイドバーでエントリを選択すると、詳細ペインにそのエントリの本文が表示されます。

- text archetype: Markdown がレンダリングされた状態で表示
- textlog archetype: 日付ごとにグループ化されたログ一覧
- todo archetype: ステータス・期日・説明（Markdown）を表示
- folder archetype: フォルダ説明（Markdown）を表示
- attachment archetype: 画像・PDF・動画・音声のプレビュー（MIME タイプに応じて）
- その他: 各 archetype の専用レンダラで表示

ドキュメント内に `![](entry:<lid>)` の形で他のエントリを **埋め込み** しておくと、詳細ビューではその参照先の内容が展開表示されます（TODO のステータス / 期日 / 説明を含むカードなど）。

### Calendar ビュー

![Calendar ビュー](asset:02_calendar-view)

期日付きの Todo を月のカレンダーに配置したビューです。

- **月の移動** — 画面上部の矢印で前月 / 次月へ、またはキーボードの矢印キー
- **Todo の表示** — 期日セルに Todo のタイトルが並びます
- **選択** — 日付セルをクリックで、その日の Todo にフォーカス
- **期日変更** — Todo を別の日付セルにドラッグすると期日が更新されます（複数選択時はまとめて移動）
- **複数選択** — `Ctrl+Click` / `Shift+Click` で複数の Todo を選び、一括で期日・ステータスを変更できます
- **アーカイブの扱い** — Calendar ビューは `showArchived` フラグに従います（既定は非表示）

### Kanban ビュー

![Kanban ビュー](asset:03_kanban-view)

Todo を「Todo」「Done」の 2 列に分けて表示するカンバンビューです。

- **Todo 列** — `status === 'open'` のアクティブな Todo
- **Done 列** — `status === 'done'` の完了済み Todo
- **状態変更** — Todo 上のチェックボックスをクリックで `open` ⇄ `done` を切り替え
- **カードのドラッグ** — カードを別の列にドロップしてステータスを変更できます（複数選択時は一括で移動）
- **キーボード操作** — 矢印キーで列内・列間を移動できます
- **アーカイブの扱い** — Kanban ビューは常にアーカイブ済み Todo を除外します

## 選択状態とビュー切替

エントリの選択状態（どのエントリを詳細ペインで見ているか）は、**ビューを切り替えても維持されます**。Detail ビューで Todo を選択してから Calendar ビューに切り替えても、同じ Todo が選択状態のままです。

複数選択も同様に維持されますが、新しいビューで対象エントリが表示されない場合は一括操作ボタンは無効になります。

## 通知（Toast）

PKC2 の非ブロッキングな通知は、画面の隅に小さな **toast** として現れます。

- **警告 toast（warn）** — 操作は成功したが注意点があるとき（例: ZIP import で衝突が検知されたとき）
- **情報 toast（info）** — 完了通知など
- **エラー toast（error）** — 操作が失敗したとき

toast は自動で数秒後に消えます。×ボタンをクリックして手動で閉じることもできます。同じ内容の toast が続けて出ると、スタックせずに 1 件に集約されます。
