# キーボードショートカット

PKC2 の主要なキーボードショートカットの一覧です。`Ctrl` は macOS では `Cmd` に読み替えてください。

## エントリ操作

| ショートカット | 機能 |
|---------------|------|
| `Ctrl+N` | 新しいエントリを作成（種類は直近のデフォルト） |
| `Ctrl+S` | 編集中のエントリを保存・確定 |
| `Escape` | 編集をキャンセル / モーダル・選択モードを閉じる |
| `Enter` | 選択中のエントリを編集モードで開く（サイドバーからの場合） |

## TextLog 固有

| ショートカット | 機能 |
|---------------|------|
| `Ctrl+Enter` | TextLog に追記を確定 |

## 入力補助

| ショートカット | 機能 |
|---------------|------|
| `Ctrl+;` | 現在の日付を `yyyy/MM/dd` 形式で挿入 |
| `Ctrl+:` | 現在の時刻を `HH:mm:ss` 形式で挿入 |

日付・時刻挿入は、text / textlog などのテキスト編集中に使えます。

## ヘルプ

| ショートカット | 機能 |
|---------------|------|
| `Ctrl+?` / `⌘+?` | ショートカットヘルプのオーバーレイ表示 / 非表示 |

`Ctrl+?`（Mac は `⌘+?`）を押すと、現在のコンテキストで使えるショートカットの一覧がオーバーレイで表示されます。迷ったらまず `Ctrl+?` を押してみてください。（`?` 単独では発火しないため、IME や Markdown 入力中でもショートカットが邪魔になりません。）

## ペイン表示の切替

| ショートカット | 機能 |
|---------------|------|
| `Ctrl+\` / `⌘+\` | 左ペイン（サイドバー）を表示 / 非表示 |
| `Ctrl+Shift+\` / `⌘+Shift+\` | 右ペイン（情報パネル）を表示 / 非表示 |

同じショートカットをもう一度押すと元に戻せます。画面を広く使いたいとき / 詳細に集中したいときに使ってください。

## キーボードだけでのサイドバー移動

マウスを使わずにエントリを渡り歩けるよう、サイドバーでは次の矢印操作が使えます。

| ショートカット | 機能 |
|---------------|------|
| `↑` / `↓` | 前後のエントリへ移動 |
| `→` | フォルダを展開する（閉じていた場合） / 展開済みなら最初の子エントリへ |
| `←` | フォルダを折りたたむ（開いていた場合） / 閉じていたら親フォルダへ戻る |
| `Enter` | 選択中のエントリを編集モードで開く |

`←` は「フォルダを閉じる」だけでなく、エントリの親フォルダにジャンプする動作にも使えます（折りたたみ状態を尊重しつつ、階層を上に戻れます）。

## Calendar / Kanban 内での移動

| 操作 | 機能 |
|------|------|
| `↑` `↓` `←` `→` | Calendar: 週・日単位で日付セル間を移動 / Kanban: 列内と列間の Todo を移動 |
| `Ctrl+↑` `Ctrl+↓` | Kanban: 現在の列内を上下移動 |
| `Enter` | 選択中のエントリを編集モードで開く |

日付セルが空の場合は自動で次の有効セルまでスキップします。月境界や列境界では移動は停止します。

## サイドバー操作

| 操作 | 機能 |
|------|------|
| クリック | エントリを選択 |
| ダブルクリック | エントリを別ウィンドウで開く（ダブルクリック動作はコンテキストに応じて編集モード開始にもなります） |
| 右クリック | コンテキストメニューを表示 |
| `Ctrl+Click` / `Cmd+Click` | 複数選択に追加 / 解除 |
| `Shift+Click` | 範囲選択 |

## 別ウィンドウ（Entry Window）で使えるショートカット

エントリをダブルクリックなどで開く **別ウィンドウ** は、メインウィンドウとは別のブラウザウィンドウとして動きます。別ウィンドウは本文編集に特化した narrowed surface なので、メインウィンドウと同じショートカットがすべて使えるわけではありません。**現時点で使えるのは本文編集に直接関係する次のものだけ** です。

| ショートカット | 動作 |
|---------------|------|
| `Ctrl+S` / `Cmd+S` | 編集中の内容を保存（`保存` ボタンと同じ動作） |
| `Escape` | 編集中なら編集をキャンセルして閲覧状態に戻す、閲覧状態でそのまま押すと別ウィンドウを閉じる |
| `Ctrl+;` / `Cmd+;` | 本文に現在日時（yyyy/MM/dd）を挿入（編集モード時のみ） |
| `Ctrl+:` / `Cmd+:` | 現在時刻（HH:mm:ss）を挿入（編集モード時のみ） |
| `Ctrl+Shift+;` / `Cmd+Shift+;` | 現在日時（yyyy/MM/dd HH:mm:ss）を挿入（編集モード時のみ） |
| `Ctrl+D` / `Cmd+D` | 日付（yy/MM/dd ddd）を挿入（編集モード時のみ） |
| `Ctrl+Shift+D` / `Cmd+Shift+D` | 日付と時刻（yy/MM/dd ddd HH:mm:ss）を挿入（編集モード時のみ） |
| `Ctrl+Shift+Alt+D` / `Cmd+Shift+Alt+D` | ISO 8601 形式の日時を挿入（編集モード時のみ） |

コピー・貼り付け・切り取り（`Ctrl+C` / `V` / `X`）、`Tab` / `Shift+Tab` などのブラウザ標準ショートカットは、別ウィンドウでも通常どおり動作します。

### 別ウィンドウで使えないショートカットとその理由

次のショートカットは **バグではなく、別ウィンドウの画面構成上そもそも対象がないため意図的に無効** になっています。

- **ペイン切替（`Ctrl+\`, `Ctrl+Shift+\`）** — 別ウィンドウにはサイドバーやメタペインが存在しません
- **ショートカット一覧（`Ctrl+?` / `Cmd+?`）** — 別ウィンドウ自体にはヘルプオーバーレイを持たせていません。必要なときはメインウィンドウ側で開いてください
- **新規テキスト作成（`Ctrl+N`）**、**TextLog 追記確定（`Ctrl+Enter`）**、**スラッシュメニュー（`/`）**、**矢印による選択移動・`Enter` での起動** — いずれもサイドバー・シェルメニュー・選択モデル・追記フォームなど、別ウィンドウが持たない面が対象です

Undo / Redo はメインウィンドウ・別ウィンドウ双方でブラウザ標準のテキストエリア undo に任せています。エントリ単位の Undo/Redo はまだ実装されていません。

開発者向けの詳細な parity 棚卸し（parity 済み / 意図的 non-parity / backlog の 3 分類）は [`docs/development/child-window-shortcut-parity-status.md`](../development/child-window-shortcut-parity-status.md) を参照してください。

## 編集モードでの一般的なテキスト操作

PKC2 はブラウザ標準のテキスト入力 UI を使っているため、次のブラウザ標準ショートカットも有効です。

| ショートカット | 機能 |
|---------------|------|
| `Ctrl+C` | コピー |
| `Ctrl+V` | 貼り付け |
| `Ctrl+X` | 切り取り |
| `Ctrl+Z` | 元に戻す |
| `Ctrl+Y` / `Ctrl+Shift+Z` | やり直し |
| `Ctrl+A` | 全選択 |

## モーダルの Escape 優先順位

`Escape` は画面内で「今いちばん手前の作業」を閉じる動作をします。同時に複数のモーダルが開いている場合、次の優先順位で閉じます。

1. TEXTLOG → TEXT のプレビュー（Slice 4）
2. TEXT → TEXTLOG のプレビュー（Slice 5）
3. TEXTLOG の選択モード（いずれのモーダルも開いていない場合）
4. Asset picker / Autocomplete など小型ポップアップ

このため、プレビューの内容をキャンセルしたいときは `Escape` を 1 回、選択モードも抜けたいときはもう 1 回、というように押すだけで状態を片付けられます。

## 注意

- ショートカットは編集モードと閲覧モードで挙動が変わる場合があります
- ブラウザや OS の予約キー（例: `Ctrl+T` は新しいタブ）は PKC2 から捕捉できません
- キーバインドは将来のバージョンで変更される可能性があります
