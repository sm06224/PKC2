# 保存と持ち出し

PKC2 のデータがどこに保存され、どうやって持ち出せるかを説明します。詳しい Export / Import / Rehydrate の手順は次章を参照してください。

## データの保存場所

PKC2 は変更を **ブラウザの IndexedDB** に自動保存します。

- **ブラウザごと・ドメインごと**: Chrome で開いた PKC2 のデータは、同じ Chrome の同じ HTML ファイル URL からしか読み書きできません
- **URL が変わると別のストア**: `file:///` で開いた場合と `https://` で開いた場合では別のストレージになります
- **ストレージのクリアで消える**: ブラウザの「サイトデータを削除」「履歴を削除」で PKC2 のデータも消える可能性があります

重要なデータは必ず Export でバックアップしてください。

## ストレージ構成

IndexedDB 内で PKC2 は次の 2 つのストアを使います。

- **containers** — エントリ・リレーション・リビジョンなどのメタデータ
- **assets** — ファイル添付の実体（base64 形式）

この分離のおかげで、添付ファイルを多用してもメタデータの読み書きは速いままです。

## 自動保存のタイミング

- 編集確定の **約 300 ミリ秒後** に IndexedDB へ書き込み
- 連続した変更はデバウンスされ、1 回の保存にまとめられます
- ブラウザタブを閉じるときに未保存の変更があれば、`pagehide` イベントで強制保存

手動保存のボタンはありません。変更したら、何もしなくてもディスクに残ります。

## データを持ち出す方法

PKC2 には 3 種類の Export があります。

| 形式 | 特徴 | 向いている場面 |
|------|------|---------------|
| **HTML Light** | テキストのみ。添付ファイル実体なし。数十 KB | メール添付、軽量共有 |
| **HTML Full** | 全データ含む。gzip 圧縮。自己完結。数 MB〜 | 完全アーカイブ、オフライン利用 |
| **ZIP Package** | 添付ファイルを生バイナリ保存。最小サイズ | バックアップ、マシン間移行、Git 管理 |

HTML Export には **Editable** と **Readonly** のモードがあります。

- **Editable** — 開いたらそのまま編集できる
- **Readonly** — 閲覧専用。**Rehydrate** ボタンで編集可能ワークスペースに昇格できる

## データを読み込む方法

- **Import** — ワークスペースで Import ボタンを押し、`.html` または `.zip` を選択。**現在のデータは完全に置き換えられます**（マージはありません）
- **Rehydrate** — Readonly HTML を直接開いている状態で Rehydrate ボタンを押す。新しい Container ID が発行され、編集可能ワークスペースとして IndexedDB に保存されます

## バックアップの推奨

- **日常のバックアップ**: 月 1 回など定期的に ZIP Package として書き出し、別のストレージ（USB メモリ、クラウドドライブなど）に保存
- **重要な編集前**: Import で大量置換する前は必ず事前に Export しておく
- **マシン移行前**: ZIP Package が最もロスなく移行できます
