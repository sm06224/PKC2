# 日常操作

エントリの作成から削除・復元、関連付けまでの基本操作を網羅します。

## エントリの作成

ツールバーの **+ Note** / **+ Todo** / **+ Form** / **+ File** のいずれかを押します。

- **+ Note** — text archetype の新規エントリ
- **+ Todo** — todo archetype の新規エントリ
- **+ Form** — form archetype の新規エントリ
- **+ File** — ファイル選択ダイアログ → attachment archetype

作成直後は編集モードに入り、すぐにタイトルと本文を入力できます。

## 編集サイクル

PKC2 の編集は次のフェーズで進みます。

1. **閲覧（ready）** — サイドバーでエントリを選択すると詳細ペインに表示
2. **編集開始（editing）** — 編集ボタン、またはダブルクリックで編集モードへ
3. **編集確定（commit）** — 保存ボタン、または `Ctrl+S` / `Cmd+S`
4. **編集キャンセル（cancel）** — Escape キー、またはキャンセルボタン

編集中は他のエントリに移動できません（編集中の変更を誤って失わないためのガードです）。

## クイック更新

いくつかの操作は編集モードに入らずに直接実行できます。

- **Todo のステータス切替** — Todo 上のチェックボックス `[ ]` / `[x]` をクリック
- **TextLog の追記** — 編集モードで `Ctrl+Enter` で確定する、または追記欄から直接追記
- **TextLog の★フラグ切替** — 追記単位の星アイコンをクリック

これらのクイック更新は本文だけを書き換え、編集フェーズには遷移しません。

## 編集中の入力補助

TEXT エントリの本文（body）を編集中に使える、小さな入力補助が 2 つあります。どちらも **現在編集中の TEXT 本文のみ** が対象で、タイトル・textlog 追記欄・form などでは動作しません。

### リンク付きコピーの貼り付け

Web ブラウザ等からリンクをコピーして TEXT 本文に貼り付けると、自動で Markdown リンク `[ラベル](URL)` に正規化されます。

- **対象**: 現在編集中の TEXT エントリの本文 textarea
- **非対象**: textlog 追記欄・別ウィンドウ内の編集・タイトル欄・form など
- **条件**: クリップボードに text/html が含まれていて、かつリンク（`<a href>`）が 1 つ以上あること
  - プレーンテキストだけのコピーでは従来通り、そのままの文字列として貼り付けられます
- **安全**: `javascript:` / `data:` などの危険な URL はリンク化されず、ラベルのみ通常テキストとして残ります

### 本文の検索・置換（Find & Replace）

編集中の TEXT 本文に対して、一括置換ダイアログを開けます。

- **開き方**: 編集モードの action bar にある `🔎 Replace` ボタン（TEXT エントリを編集中のときにだけ表示されます）
- **入力**:
  - **Find** — 検索文字列
  - **Replace with** — 置換文字列
- **オプション**
  - **Regex** — JavaScript の正規表現として Find を解釈。`$1` などの back-reference が使えます
  - **Case sensitive** — 大文字小文字を区別
  - **Selection only** — ダイアログを開く **前** に本文 textarea で文字列を選択しておくと、検索・置換の対象が選択範囲内に限定されます。選択せずに開いた場合はこのオプションは灰色になり使えません。選び直したい場合はいったんダイアログを閉じて、選択し直してから再度開いてください
- **プレビュー**: 状態行に「N match(es) will be replaced.」とヒット件数がリアルタイムに表示されます
- **Apply**: ヒット件数が 1 以上のときだけ有効。押すと一括で置換され、件数が 0 に更新されます（ダイアログは開いたまま、続けて別の置換ができます）
- **閉じ方**: Close ボタン / ダイアログ外側クリック / `Escape`。閉じた後は本文 textarea にフォーカスが戻ります
- **Enter キー**: Find / Replace 入力欄で Enter を押すと、Apply が有効なときは即時実行されます

#### Apply が無効になる条件

- Find 欄が空
- Regex が有効で、正規表現として無効な文字列（状態行に赤文字でエラーメッセージが表示されます）
- ヒット 0 件（`No matches in current entry.`）

#### 対象外

- タイトル / source_url / tags の置換
- textlog / form / attachment / folder の本文
- **readonly** ワークスペース（そもそも編集モードに入れないため `🔎 Replace` ボタンは出ません）
- **historical revision** を閲覧中（編集モードに入れないため同上）
- 全エントリ一括置換・複数エントリ置換

## 削除と復元

### 削除

1. 削除したいエントリを選択
2. 削除ボタン、または複数選択して一括削除
3. 確認ダイアログで **OK**

削除されたエントリはリビジョン履歴から復元できます（完全消去ではありません）。

### 復元

1. リビジョン履歴を開く
2. 復元したい時点のスナップショットを選択
3. 復元ボタン

復元は「元の状態に戻す」ではなく「スナップショットの内容で今を上書きする」動作です。復元前の状態も別のリビジョンとして自動で記録されるので、復元を取り消したくなっても履歴は失われません。

**制約**: リビジョンに記録された archetype と現在の archetype が違う場合は、復元は拒否されます（TEXT エントリに TODO の JSON を書き戻してしまうような事故を防ぐためのガードです）。

### ゴミ箱の完全消去

リビジョン履歴も含めて本当に消したい場合は **Purge Trash** を使います。これは取り消せない操作です。

## 複数選択と一括操作（Multi-select）

サイドバー・Calendar・Kanban のどのビューからでも、複数のエントリをまとめて操作できます。

### 選択方法

| 操作 | 意味 |
|------|------|
| クリック | そのエントリだけを選択（他は解除） |
| `Ctrl+Click` / `Cmd+Click` | 個別に追加 / 解除 |
| `Shift+Click` | 直前の選択から範囲選択 |
| `Escape` | 複数選択をクリア |

選択中のエントリには強調表示が付きます。選択数はツールバーの一括操作ボタンの有効・無効で確認できます。

### 使える一括操作

- **一括削除** — 選択中のエントリをまとめて削除
- **フォルダへ一括移動** — 選択中のエントリを指定フォルダの子にする
- **ルートへ一括移動** — 選択中のエントリをフォルダ配下から外す
- **一括ステータス切替**（Kanban / Calendar でのみ）— 選択中の Todo を `open` / `done` に揃える
- **一括期日設定 / クリア**（Kanban / Calendar でのみ）— 選択中の Todo の期日を同時に書き換え

### 注意点

- ビューを切り替えても選択は維持されますが、対象のエントリが新しいビューに表示されない場合は操作ボタンが無効になります
- フィルタや並び順を変えると、視覚的な並びと選択の順序が一致しなくなることがあります。一括削除の前に対象件数を確認してください

## ドラッグ&ドロップ（Drag & Drop）

エントリの並び替えや関係付けをマウス操作で行えます。用途ごとに 3 つの系統があります。

### サイドバーでのツリー移動

- サイドバーでエントリを別フォルダの上にドロップすると、そのフォルダの子になります（`structural` relation が張り替えられます）
- ルート領域にドロップすればフォルダ配下から外せます
- folder 同士のドロップでフォルダの親子関係も変えられます

### Kanban でのステータス変更

- Todo カードを別の列（Open / Done）にドロップすると、そのままステータスが切り替わります
- 複数選択した状態でドラッグすると、選択中の Todo すべてが一度に移動します

### Calendar での期日変更

- Todo を別の日付セルにドロップすると、その日付が期日として設定されます
- 複数選択状態でのドラッグも同様に全件まとめて期日が変わります

### 注意点

- ドラッグ中はドラッグゴーストに件数バッジが表示され、何件動くかが見えます
- DnD でのフォルダ変更は `Relation` の張り替えであり、エントリ本体の内容は変わりません
- ドロップ先が意図と違っていたら、そのまま Escape でキャンセルできます

## コンテキストメニュー

サイドバーのエントリを右クリックするとコンテキストメニューが開き、よく使う操作にすぐアクセスできます。

- エントリの編集 / 複製 / 削除
- フォルダへの移動
- 別ウィンドウで開く（後述）
- このエントリを単独で持ち出す（selected-only export）

コンテキストメニューから開いた操作は、ツールバーのボタンから行うものと機能的に同じです。

## 別ウィンドウ表示（Entry Window）

エントリを別のブラウザウィンドウで開いて、メインのウィンドウと並べて作業できます。

- **開き方**: エントリをダブルクリック（コンテキストメニューや `Enter` キーからも開けます）
- **用途**: 参照用に常時表示しておきたいメモ、編集とプレビューを並べて見たいドキュメントなど
- **追従**: メイン側で対象エントリや、別エントリに埋め込まれている TODO などが更新されると、別ウィンドウの表示も自動で更新されます
- **閉じ方**: ウィンドウをそのまま閉じます。メインワークスペースの状態には影響しません

### 注意点

- 別ウィンドウは **同じタブ内のポップアップ** としてブラウザに扱われます。ブラウザのポップアップブロックが有効だと開けないことがあります
- ブラウザを丸ごと閉じると、別ウィンドウも閉じます
- 別ウィンドウで編集を確定すると、メインのワークスペースに保存されます（メイン側での編集と競合する場合は、確定が後勝ちになります）

## TEXT と TEXTLOG を変換する

TEXT（Markdown ドキュメント）と TEXTLOG（時系列ログ）は、中身を保ったまま相互に変換できます。

### TEXTLOG → TEXT

TEXTLOG のログ行を選んで、1 つの Markdown ドキュメントにまとめます。

1. TEXTLOG を開く
2. **ログを選択** ボタンで選択モードに入る
3. まとめたいログにチェックを入れる
4. **→ TEXT に変換** で preview を表示し、内容を確認
5. **作成** で新しい TEXT エントリが作られる（元の TEXTLOG は変更されません）

### TEXT → TEXTLOG

TEXT の本文を章ごとに区切って、新しい TEXTLOG を作ります。

1. TEXT を開く
2. **→ TEXTLOG に変換** ボタンで preview を表示
3. 区切り方を選ぶ
    - **ATX heading (#, ##, ###)** — 見出しごとに 1 ログ
    - **Horizontal rule (---)** — `---` ごとに 1 ログ
4. タイトルを必要に応じて編集
5. **作成** で新しい TEXTLOG が作られる

### 注意点

- preview を開いている間に別のエントリを選んだり編集モードに入ったりすると、preview は自動で閉じます
- 変換結果には「どの TEXT / TEXTLOG から生まれたか」を示すメタログ / メタリンクが自動で付きます
- 元データは書き換えられません。同じソースから何度でも変換できます

## 一括読み込み（Batch Import）

複数のテキストファイルや Markdown ファイルを、まとめて 1 回の操作で PKC2 に取り込めます。

1. **Batch Import** ボタンからファイル群を選択
2. 読み込み結果が **プレビュー** として表示される
3. 各ファイルを個別にチェックして、本当に取り込むものを選ぶ
4. **取り込み先フォルダ** を任意に指定（未指定ならルート）
5. **Confirm Import** で取り込み実行

- 取り込みは **追加** 操作です。既存のエントリは消えません（通常の Import とは違います）
- 取り込み後は何件成功 / 失敗したかの要約が通知されます

## 関連付け（Relations）

エントリ同士を 4 種類の関係で結びつけられます。

| 種別 | 意味 | 例 |
|------|------|-----|
| **structural** | 親子関係（フォルダ所属） | folder → 子エントリ |
| **categorical** | カテゴリ・タグ | 「技術メモ」タグ |
| **semantic** | 意味的参照 | 「関連するエントリ」 |
| **temporal** | 時間順 | 「前のエントリ → 次のエントリ」 |

タグは `categorical` relation の実装です。関連付けの作成・削除は各エントリの関連情報パネルから行います。

## リビジョン履歴

編集を確定するたびに、編集前の状態がリビジョンとして自動保存されます。

- **閲覧**: リビジョン履歴パネルで過去のスナップショットを一覧
- **差分**: 現在の本文との違いを確認
- **復元**: 過去のリビジョンを現在の状態として上書き

リビジョンは Container に保存されるため、Export にも含まれます。

## アーカイブ表示

完了した Todo は **アーカイブ** することで日常の視界から隠せます。

- **アーカイブする**: Todo の詳細でアーカイブボタン
- **表示切替**: `showArchived` フラグで一覧に出すかを切り替え
- **Calendar ビュー**: `showArchived` に従う
- **Kanban ビュー**: 常に除外

## 自動保存について

PKC2 は変更のたびに **約 300 ミリ秒後** に IndexedDB へ自動保存します。

- 連続した編集はまとめて 1 回の保存にデバウンスされます
- ブラウザタブを閉じるときに未保存の変更があれば、ブラウザの `pagehide` イベントで強制的に保存されます
- 手動保存ボタンはありません
