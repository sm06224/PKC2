## 反省点(同じ轍を踏まないために)

前セッションで Claude(私)は以下の致命的な瑕疵を犯した。次セッションでは絶対に再発させない:

1. **生成(HTML / state mutation)を「動く」と扱った**。`renderMarkdown` が anchor を吐いた / `selectionStart` が変わった、で「ユーザー実機で動く」と書いたが、ユーザーが実際に見るピクセルと操作の経路を経た検証ではなかった。`locator.click()` も programmatic で OS event を経由しない。
2. **ユーザーが「動かない」と言う度にこちらが推測で進めた**。AppState / dispatch / DOM / 環境を**ユーザーが 1 アクションで吐ける**導線が無いまま自然言語ヒアリングを続け、ユーザー負担を強いた。

ユーザーは「描画と生成を同じものとして受け取っている」「表示と操作を一致させなきゃいい物にならない」「ユーザーがデバッグ報告するための導線がない」と明確に指摘した。**この 3 つは reform の核**。
